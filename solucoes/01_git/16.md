## Problema

16 - Qual a diferença entre um _merge fast forward_ e um _merge commit_? Em
qual destes _merges_ é possível haver um _merge conflict_?

## Soluções

### Solução 1

No nosso git existe dois ramos, o _master_ e o _disciple.

```merge fast foward```
Se o ramo _master_ não tiver divergências com o ramo _disciple_, em vez de criar um novo commit, irá apontar para o ultimo commit efectuado pelo o ramo _disciple_, visto que não houve qualquer alteração no ramo _master_, toda a serie de commits fica linear.

```merge commit```
Se o _master_ tiver divergências desde a criação do ramo _disciple_, ambos os ramos apontam para commits diferentes. O git neste caso irá aplicar as mundanças introduzidas pelo os dois ramos em um novo commit.

Às vezes, este processo não funciona correctamente. Caso tenha sido alterado o mesmo ficheiro, nas mesmas linhas em ambos os ramos, o Git não irá executar o merge, sendo necessário efectuar a respectiva correção.

*Solução proposta por [André Santos](https://github.com/Snigy24)*
